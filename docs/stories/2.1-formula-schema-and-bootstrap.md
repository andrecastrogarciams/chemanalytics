## Story

**As a** administrador do sistema,  
**I want** um modelo persistente de fórmulas, versões e itens com bootstrap inicial,  
**so that** o sistema substitua a planilha atual com rastreabilidade de vigência.

## Acceptance Criteria

1. O banco contém entidades para fórmula, versão e item químico com relacionamentos coerentes.
2. O modelo impede sobreposição de vigência para o mesmo par `CODPRO + CODDER`.
3. Existe mecanismo de importação inicial da planilha com relatório de sucesso e rejeições.
4. Fórmulas históricas não são fisicamente excluídas.
5. Há testes cobrindo regras de vigência, duplicidade e bootstrap inicial.

## Testing

- Testes de banco para constraints de vigência.
- Testes de integração para importação inicial.
- Cenários com linhas válidas, inválidas e duplicadas.
- Validado em 2026-03-18:
  - `python backend\manage.py test apps.formulas`
  - `python backend\manage.py bootstrap_formulas backend\apps\formulas\fixtures\sample_formulas.csv --format=json`
  - `python backend\manage.py bootstrap_formulas imports\PREVISAO_CONSUMO_PQ.xlsx --format=json`
  - `python backend\manage.py check`
- Cobertura implementada:
  - unicidade de fórmula por `codpro + codder`
  - fechamento automático da versão anterior aberta
  - bloqueio de sobreposição de vigência
  - bloqueio de item químico duplicado por versão
  - bootstrap com relatório de criados e rejeições
  - suporte a CSV e XLSX no comando de bootstrap
  - tratamento explícito de item incompleto quando `%` vier ausente na fonte
- Resultado da planilha real:
  - importação inicial criou 13 fórmulas, 14 versões e 342 itens completos
  - reprocessamento posterior incorporou 10 itens incompletos antes rejeitados
  - `%` vazio/nulo agora gera item com `is_incomplete = true` e motivo explícito
  - `rejected_rows` ficou zerado para esse caso de dado incompleto conhecido
