# Epic 4: Operability and Rollout

## Objetivo

Fechar o MVP com administração de usuários, documentação operacional e capacidade mínima de deploy/rollback.

## Valor de negócio

Permite uso real em ambiente interno com menor risco operacional e menor dependência tácita do desenvolvedor original.

## Escopo

- Gestão de usuários
- Manual rápido
- Guia de instalação/deploy
- Runbook operacional

## Fora de escopo

- Automação avançada de CI/CD
- Alta disponibilidade
- Observabilidade enterprise

## Dependências

- Epics 1, 2 e 3 concluídos

## Stories

1. `4.1` User management
2. `4.2` Operational docs and deploy

## Definition of Done do épico

- Administração de usuários está disponível para perfil adequado
- Documentação mínima citada na spec existe e é utilizável
- Procedimentos de deploy e rollback estão descritos
