## Story

**As a** usuario revisor,  
**I want** revisar manualmente divergencias com justificativa obrigatoria,  
**so that** excecoes operacionais fiquem registradas sem perder rastreabilidade.

## Acceptance Criteria

1. Apenas perfil autorizado consegue executar revisao manual.
2. A revisao exige justificativa antes de persistir alteracao de status.
3. O sistema registra usuario, data/hora, status anterior, novo status e justificativa.
4. O historico de revisao nao sobrescreve trilhas anteriores.
5. Ha testes cobrindo autorizacao, obrigatoriedade de justificativa e trilha de auditoria.

## Dev Notes

- O foco e preservar governanca, nao editar silenciosamente o resultado historico.
- O modelo `ReconciliationItemReview` guarda cada revisao como evento imutavel.
- O item continua mantendo `status_calculated`, enquanto `status_reviewed` e `status_final` refletem o ultimo override manual.
- O lote e recalculado a partir dos `status_final` dos itens sempre que uma revisao e criada.
- Endpoint entregue:
- `POST /api/v1/reconciliation/items/{item_id}/reviews`
- A trilha de auditoria tambem fica visivel no detalhe do lote via `GET /api/v1/reconciliation/lots/{lot_id}`.
- Referencias: `docs/PRD.md#131`, `docs/PRD.md#241`, `docs/SPEC.md#1043`.
