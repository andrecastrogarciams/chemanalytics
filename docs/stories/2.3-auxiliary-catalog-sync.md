# Story 2.3: Auxiliary catalog sync

## Status

Review

## Executor Assignment

- executor: `@dev`
- quality_gate: `@architect`
- quality_gate_tools: `["review", "test"]`

## Story

**As a** operador do sistema,  
**I want** sincronizar artigos, derivações e produtos químicos a partir do Oracle,  
**so that** os cadastros de fórmula usem listas consistentes com o ambiente operacional.

## Acceptance Criteria

1. O sistema sincroniza catálogos auxiliares definidos na spec a partir do Oracle.
2. A sincronização pode ser executada manualmente e por agenda.
3. O cache local mantém códigos e descrições necessários para uso da aplicação.
4. Falhas de sincronização ficam registradas em log com motivo objetivo.
5. Há testes cobrindo sucesso, timeout e falha de integração.
