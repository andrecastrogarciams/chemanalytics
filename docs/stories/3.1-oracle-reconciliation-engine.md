## Story

**As a** usuario do almoxarifado quimico,  
**I want** executar conferencias por periodo usando dados do Oracle e formula vigente,  
**so that** eu consiga comparar previsto versus realizado com base confiavel.

## Acceptance Criteria

1. O usuario inicia uma conferencia informando filtros minimos definidos no produto.
2. O sistema consulta dados operacionais necessarios no Oracle em modo somente leitura.
3. Para cada lote/item, o previsto e calculado com base na formula vigente correta.
4. O motor classifica divergencias e inconsistencias conforme taxonomia oficial.
5. A execucao persiste um historico congelado com dados suficientes para auditoria futura.

## Dev Notes

- Esta e a story de maior risco e maior valor do MVP.
- O historico persistido nao pode depender de releitura posterior do Oracle.
- Referencias: `docs/PRD.md#219`, `docs/PRD.md#229`, `docs/SPEC.md#978`, `docs/SPEC.md#1197`, `docs/SPEC.md#1241`.
- A implementacao atual usa `OracleReconciliationAdapter` com fixture JSON local para validacao offline, preservando o contrato de leitura read-only.
- A reconciliacao congela `run`, `lotes` e `itens` com previsto, realizado, desvio, taxonomia de inconsistencia e versao de formula resolvida.
- O adapter normaliza datas ISO antes da resolucao da formula vigente para evitar comparacao incorreta com strings.
